# 🚀 クイックスタート - 自動トレーニング

## 📋 概要

このガイドでは、**3つのコマンド**だけでクラブヘッド検出モデルを自動的にトレーニングできます。

### 🎯 目標
- ✅ データ拡張: 66枚 → 266枚（200枚追加）
- ✅ トレーニング: 100エポック
- ✅ 信頼度: 50%以上

### ⏱️ 所要時間
- セットアップ: 5〜10分
- トレーニング: 30分〜2時間（GPU使用時）

## 🔧 カスタマイズ

### データ拡張枚数を変更

`auto_augment_and_train.py` の以下の行を編集：

```python
# 200枚追加 → 300枚追加に変更
augmenter.augment_dataset(target_count=300)
```

### エポック数を変更

```python
# 100エポック → 150エポックに変更
trainer.train_model(epochs=150, batch=16)
```

### バッチサイズを変更（メモリ不足時）

```python
# 16 → 8に変更
trainer.train_model(epochs=100, batch=8)
```

### より大きなモデルを使用

```python
# yolov8n → yolov8s に変更（精度↑、速度↓）
model = YOLO('yolov8s.pt')
```

## 📈 学習曲線の見方

トレーニング完了後、`runs/detect/clubhead_high_confidence/results.png` を確認：

### 良好な学習曲線

```
Loss（損失）
  ↓ 右肩下がり（良い）
  
mAP（精度）
  ↑ 右肩上がり（良い）
  
Precision/Recall
  → 安定して高い値（良い）
```

### 問題のある学習曲線

```
Loss
  ～ 振動が激しい → バッチサイズを調整
  
mAP
  → 横ばい → エポック数増加 or データ追加
  
Overfitting（過学習）
  train_loss ↓ val_loss ↑ → データ拡張強化
```

## 🆘 トラブルシューティング

### Q: メモリ不足エラー

```
CUDA out of memory
```

**解決策**:
```python
# バッチサイズを減らす
trainer.train_model(epochs=100, batch=8)  # 16 → 8
```

---

### Q: 精度が50%に届かない

**解決策**:
1. データ追加（300枚に増やす）
2. エポック数増加（150に増やす）
3. より大きなモデル（yolov8s）

---

### Q: トレーニングが遅い

**GPU確認**:
```bash
python -c "import torch; print(torch.cuda.is_available())"
```

- `True` → GPU使用中（正常）
- `False` → CPU使用中（遅い）

**GPU使用時の所要時間**:
- 30分〜1時間

**CPU使用時の所要時間**:
- 2〜4時間
